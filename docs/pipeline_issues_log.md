## 2. 코드 수정 필요

### 2-1. 중복 컬럼 `.1` 접미사 처리

**증상:** `서버 이름.1`, `에이전트 IP.1`, `운영 부서.1`, `플랫폼 부서.1`, `서비스.1` 등
**원인:** Excel에 동일 컬럼명이 2개 있을 때 pandas가 자동으로 `.N` 접미사 부여
**영향:** column_normalizer가 `.1` 붙은 이름을 인식 못해 컬럼 drop됨
**수정 위치:** `src/data/column_normalizer.py` — normalize() 호출 전 `.1`, `.2` 등 접미사 제거 로직 추가

## 4. 유효 시트 없음 → fallback 발생 파일

아래 파일들은 메인 데이터 시트가 아닌 요약 시트를 fallback으로 읽음.
컬럼 구조가 달라 대부분의 컬럼이 미매핑 처리됨 (데이터 영향 없음).

- `오탐 취합 보고 자료_3월_xlsx`
- `오탐 취합 보고 자료_3월_CTO.xlsx`
- `오탐 취합 보고 자료_3월_NW.xlsx`
- `오탐 취합 보고 자료_4월_xlsx`
- `오탐 취합 보고 자료_4월_NW.xlsx`
- `오탐 취합 보고 자료_5월_품질.xlsx`
- `정탐 취합 보고 자료_3월_기업부문.xlsx`
- `정탐 취합 보고 자료_9월_기업.xlsx`

**원인:** 해당 파일들의 메인 데이터 시트에 `서버이름` 계열 컬럼이 없어 유효 시트 판별 실패
**조치 옵션:** ingestion_config.yaml에서 해당 파일 패턴 exclude 또는 무시
